# Тестовое задание для GREEN-API (DevOps/SysAdmin)

Данный проект представляет собой реализацию клиентской части для работы с методами [GREEN-API](https://green-api.com).

## 🛠 Стек технологий
* **Frontend:** HTML5, JavaScript (Fetch API)
* **Web Server:** Nginx (на базе образа Alpine)
* **Infrastructure:** Docker, VPS (Self-hosted)

## 🚀 Функционал
Реализован интерфейс для выполнения следующих требований задания:
* **getSettings** — получение настроек инстанса.
* **getStateInstance** — получение состояния инстанса.
* **sendMessage** — отправка текстового сообщения.
* **sendFileByUrl** — отправка файла по прямой ссылке.

**Безопасность:** Параметры подключения (`idInstance`, `apiTokenInstance`) вводятся пользователем в интерфейсе и не сохраняются в коде проекта.

## 📦 Развертывание (Deployment)

Проект полностью подготовлен для работы в Docker-контейнере и развернут на внешнем VPS сервере.

### Сборка и запуск локально:
1. Склонируйте репозиторий:
    ```bash
    git clone https://github.com/kijyra/gapi.git
    cd gapi
    ```
2. Соберите Docker-образ:
    ```bash
    docker build -t gapi .
    ```
3. Запустите контейнер:
    ```bash
    docker run -d -p 8080:80 --name GreenAPI gapi
    ```
После запуска интерфейс будет доступен по адресу: http://localhost:8080

### Особенности конфигурации:

* **Reverse Proxy & API Gateway:** Nginx настроен не только как веб-сервер, но и как прокси-сервер для взаимодействия с GREEN-API.
* **Оптимизация:** Использование легковесного образа `nginx:alpine` и кастомного конфига для маршрутизации трафика.
* **Docker:** Приложение полностью инкапсулировано, что гарантирует идентичность работы в local и prod окружениях.
* **VPS**: Деплой произведен на удаленный сервер, доступ к приложению настроен через порт 8080.
